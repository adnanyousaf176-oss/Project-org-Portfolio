**Dawat-e-Islami Website Testing**
This repository contains the Test Plan and related testing documentation for the newly developed version of the Dawat-e-Islami website. The aim of this project is to ensure that the website is fully functional, user-friendly, and meets quality standards before public release.
**Project Overview**
The project involves the development and testing of the Dawat-e-Islami website with the following key modules and functionalities:
Home Page
Media Section (images, videos, documents, PDFs)
News Updates Module
Event Module
Language Toggle (Urdu/English)
Slider & Banners
About Us Dynamic Page
Organizational Hierarchy integrated with PMS hierarchy
Organizational Structure Module
Document Upload Functionality
Responsive Design (Desktop, Tablet, Mobile)
Testing Objectives
The main objectives of testing are:
Verify that all website features function as expected.
Ensure the user interface is responsive and user-friendly across all major devices and browsers.
Detect and resolve bugs prior to release.
Ensure data accuracy, language correctness (Urdu/English), and performance standards.
Testing Types
The following types of testing were performed:
Functional Testing – Verify all modules and features work as expected.
UI / UX Testing – Check responsiveness, layout, sliders, banners, forms, and language toggle.
Content & Data Validation Testing – Validate text (Urdu/English), media display, and organizational data accuracy.
API Testing – Test backend APIs using Postman for data retrieval, submission, and integration.
Performance Testing – Test website load, stress, and response times using JMeter.
Cross-Browser Testing – Ensure compatibility across major browsers (Chrome, Firefox, Edge, Safari).
Bug Tracking & Regression Testing – Log bugs in Excel and re-test after fixes.
User Acceptance Testing (UAT) – Conducted with end users or stakeholders to verify that the website meets real-world business requirements and expectations.
Tools Used
Excel – For writing test cases and logging bug reports.
Postman – For API testing.
JMeter – For performance and load testing.
Scope of Testing
Testing covered all major functional and non-functional aspects of the website, including:
Functional verification of each module.
User interface responsiveness across devices.
Accuracy of content and media.
Language toggle correctness.
Upload and download of documents.
Performance under load conditions.
UAT feedback and validation from stakeholders.
Deliverables
Detailed test cases and results.
Bug reports logged in Excel.
API testing results via Postman.
Performance testing reports via JMeter.
UAT feedback and sign-off documentation.
